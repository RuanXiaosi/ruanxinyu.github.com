我对个人博客的要求

我对个人博客有如下几点要求，因此最终选定通过Hexo+GitHub Pages来搭建。

  1. 存在独立域名，别人能通过互联网访问
  2. 通过Git进行管理，支持MarkDown语法
  3. 不需要自己购买主机维护环境
  4. 支持评论分享等功能

配置Hexo

GitHub Pages相当于一个静态网站，您需要将您的博客全部转换为html页面才可以，当然，这个繁琐的过程不需要我们自己去完成，当前有两个框架可以做这个事情：Jekyll和Hexo，下面我就介绍一下如何通过Hexo快速的发表自己的文章。

安装Hexo

Hexo是一个NodeJS实现的博客框架，官方文档地址为： https://hexo.io/zh-cn/docs/, 文档存在中文哦，给力吧！下面说一下使用过程：

  1. 执行npm install -g hexo-cli命令安装Hexo（在安装Hexo之前，请您自行安装NodeJS和NPM）

  2. 执行hexo init xxx创建一个博客

  3. 执行npm install hexo-server --save安装server工具，然后执行hexo server，然后访问http://localhost:4000/网址可在本地访问你的博客，该方式主要用于调试。

  4. 执行hexo new xxx创建一篇博文，博文的文件存放在source\_posts目录下，编辑文档内容，刷新本地网址即可看到你更新的内容。

  5. 在将博文推送到GitHub Pages之前，执行npm install hexo-deployer-git --save安装部署工具，然后将根目录下的_config.yml文件中的如下内容更改为您的GitHub Pages的仓库地址，注意是git@开头的地址

  6. 执行hexo deploy， Hexo会自动将您的博文推送到GitHub Pages，访问您的GitHubPages就可以看到你更新的文章内容。
