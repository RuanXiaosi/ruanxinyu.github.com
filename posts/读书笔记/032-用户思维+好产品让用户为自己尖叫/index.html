    3. ( 读书笔记 | 2019-03-11 17:24:21)

不要打造更好的x，要打造更好的x用户，好产品就是让用户拥有成长型思维模式和持续学习能力，后用户体验，这是个让人眼前一亮的新视角。刻意练习原来要这么玩；感性接触真是刷新了我的认知；用户成长模型让成长直观可见，这本书值得一看。

成就卓越用户

好产品就是让用户拥有成长型思维模式和持续学习能力，不要打造更好的x，要打造更好的x用户

=> 可持续的愿望可以驱动产品成功
=> 可持续成功的产品都是由推荐产生的
=> 诚实，不作假的口碑传播可以驱动愿望
=> 用户表现卓越驱动口碑传播
=> 在工具帮助用户所做的事情上（而不是工具的使用）表现卓越，在更大的应用场景中表现卓越
=> 表现卓越不只是体现在成果上，意味着更高的清晰度
=> 表现卓越意味着用户收获卓越的成果和生意
=> 不要打造更好的x，要打造更好的x用户

后用户体验：

我们的成功是由用户体验之后所发生的那些事情驱动的。为后用户体验的用户体验而设计意味着，不仅要为你的用户设计，而且要为你的用户的用户设计。

想象一下，你的竞争优势不再是你如何与竟争对手相互较量，而是你的用户如何与竞争对手的用户相互较量。帮助一小部分用户在一个更大的应用场景中获得显著成功，与此同时，让口碑传播和事实传播成为这一事件的天然效应。

用户之旅：

我们需要关注两条用户成长曲线：一条针对我们的产品（工具），一条针对这个产品所在的应用场景。

用户成长曲线需要设定里程碑，有两个关键环节：入门线和卓越线。这里有两个问题：

  1. 在进入入门线之前属于困难期，在第一次接触和产品升级时都会进入到这个区间
  2. 用户跨越入门线之后可能进入停滞区，不再进步，进入自动模式（对于你的产品，服务和应用场景中，自动模式是什么？）

怎样帮助用户沿着成长曲线向右上方移动呢？我们必须做好两件事情：帮助他们持续不断的构建技能，清晰度和能力；帮助他们保持渴望成功的愿望。

构建技能

为工具和应用场景定义卓越：为了帮助我们的用户不断进步，最重要的是为表现或成果定义相应的条件和指标，任何条件和指标都行，即使他们不符合完美的客观测量标准。

那些成为专家的人与那些想要成为专家但是没能成功的人区别是什么？

  1. 专家练习的更好，练习活动符合刻意练习的标准和要求
  2. 专家通过大量高质量的接触和反馈活动，获得了更深层次的感性知识和技能。

刻意练习

刻意练习标准：完成一项细粒度的任务，经过1~3组45~90分钟的练习，就能达到95%的精通度。请记住我们在平庸阶段进行的练习越多，对平庸技能强化就越大，练习造成的影响是永久性的。

  1. 专家避免陷入中等技能困境：处于精通状态（也称无意识状态或自动状态）的技能，经常导致中等技能困境。那些长时间停留在自动或潜意识状态的技能将会慢慢退化，即使我们经常使用它们也是一样。
  2. 一次刻意练习一件事：通常情况下刻意练习是有意识的，从不能做到精通构建技能的最佳途径。大多数人在大多数的专业能力，成长曲线上遇到的最大问题就是阶段B堆积了太多东西。
  3. 更细粒度的子技能学习法： 刻意练习，有助于防止陷入平庸，这是因为精通半个技能，完胜一堆半生不熟的技能。在一些领域，初学者需要一些半生不熟的技能。在你的领域起步阶段的最低技能要求是什么？

感性接触

如何设计高质量的感性接触活动呢？使用大量的，高质量的表面有差异，但性质完全相同的实力。如果训练失败，最可能的原因是：

  1. 缺乏足够的实力
  2. 实力的多样化程度不够
  3. 接触与反馈的间隔时间太长
  4. 特征或模式过于细微，难以察觉

如何做？

  1. 你只需要将大量的高质量的实例在很短的时间内展现给你的用户
  2. 为你的用户创建最小可行感性接触体验
  3. 当你确实需要展示错误或不良的实例时营造不适感
  4. 学习鉴别不良实例的最佳办法就是学习优秀实力的潜藏模式

帮助用户前进

帮助用户前进包括两个部分：排除阻碍；成长路径与回报。

排除阻碍

关键的问题不是什么吸引他们前进，而是什么让他们停了下来？

  1. 当用户刚刚起步时，帮助他们向前推进的最佳办法就是集中精力排除那些阻挠他们进步的障碍。
  2. 解决这一问题的办法不是填平鸿沟，而是设法使它变得无足轻重。
  3. 让他们继续前进的秘诀就是：当他们遭遇困难时，承认这些困难。
  4. 比劣质用户手册更糟糕的情况是用户以为这份手册没有问题，其他人都可以很好的使用它。
  5. 针对干扰因素的解决方案就是预测与补偿。预测他们最可能做出的表情和最可能提出的问题。为那些他们无法告知你的体验以及更重要的为你无法给予他们的关注和反馈给予补偿。

进步与回报

成为专家的好处，不能仅仅体现在最后阶段。为了帮助用户不断渴望进步，应该给予他们：

  1. 一份描述前进路径的指南，帮助他们了解当前所处的阶段，创造回报循环
  2. 一些想法和工具帮助他们尽早尽可能的频繁的利用当前掌握的技能，触发内在激励体验

成长路径图关注做什么，而不是学什么。大多数专家倾向于教授那些最容易表述的内容，而不是那些对于改善表现最有价值的主题。专家型老师更关注知识和原理常常忽视更深层次的核心感知模式，以及最有可能创造出真正成果的实践和经验。按照正确的方法做正确的事情才是有效的成长路径，即使它并不是最佳路径。

内部动机：高清晰度（使用领域特定的专业术语进行交流，不仅有用，而且也是一种激励）和心流。

维护认知资源

减少资源泄漏：

认知资源是优先的，确保用户把稀缺，易耗的认知资源用在正确的事上，用户在认知资源管理上的核心任务就是减少资源泄漏。

  1. 如果你的大脑相信某物或某人针对未完成的认知任务，有一个可靠的应对计划，他就无需花费资源担心这项任务。
  2. 要想减少认知泄露，就应该把认知工作委托给外部世界。这样他就不会停留在用户的大脑中。
  3. 不要让用户记忆
  4. 设法让正确的事情成为最可能做的事情，让正确的行为变得自然和明显
  5. 不要让用户选择
  6. 帮助用户内化技能，每次精通一个小技能
  7. 传授使用技巧，把练习活动化繁为简
  8. 帮助用户处理琐碎但重要的事情，比如持续的周而复始的提醒
  9. 减少对意志力的需求，帮他们构建习惯
  10. 帮助他们体验内在激励，帮助他们的大脑集中注意力

大脑的垃圾过滤器：

  1. 利用场景说服大脑（愿景）
  2. 使用为什么？那又怎么样？谁会在意的策略？（多问为什么）
  3. 应对大脑的垃圾过滤器的最佳办法就是：减少需要通过过滤器的事项
  4. 利用技能映射验证知识的有用性
